# notebooklm-software-architecture-tidy-first

## 📌 Contexto

Este projeto foi desenvolvido como parte de um desafio prático da DIO com foco no uso do NotebookLM como ferramenta de aprendizado ativo.

O tema escolhido foi Arquitetura de Software, com base no livro "Tidy First?" de Kent Beck, explorando princípios de organização de código, refatoração e tomada de decisão incremental.

## 🎯 Objetivos

- Consolidar conceitos fundamentais de arquitetura de software
- Utilizar IA (NotebookLM) como copiloto de estudo
- Desenvolver pensamento crítico sobre design de código
- Criar um material reutilizável (miniguia + prompts)
- Documentar o processo de aprendizado (incluindo erros)
